# MVP-003 — File UI and Syntax Foundation

## Objective

Add discoverability and first syntax support.

## Scope

- menu bar model
- File menu
- Edit menu
- Search menu
- View menu
- Tools menu
- Help menu
- open file prompt
- save as prompt
- basic file browser
- line numbers
- whitespace display toggle
- tab insertion setting: literal tab, 2 spaces, or 4 spaces
- native syntax token model
- simple built-in highlighters
- nano syntax importer tier 1

## Acceptance criteria

- menus can be opened with Alt shortcuts
- open/save-as works without command-line path
- native highlighting model is renderer-independent
- nano syntax parser can load simple syntax rules
- all MVP-003 behavior configures, builds, and tests on macOS, Linux, and Windows

## Closeout notes

Implemented in local milestone commit for MVP-003:

- Added model-driven File, Edit, Search, View, Tools, and Help menus.
- Added Alt shortcuts for menu opening on POSIX escape parsing and Windows console input.
- Added open and save-as prompt modes that work from an unnamed startup document.
- Added a portable basic file browser model with parent entry, directories-first sorting, files second, selection, and activation results.
- Added line-number and whitespace-display settings.
- Added tab insertion modes: literal tab, 2 spaces, and 4 spaces.
- Added renderer-independent native syntax token model.
- Added a simple built-in C-like highlighter.
- Added Tier 1 nano syntax import for `syntax`, `color`, and `icolor` lines with simple filename suffix patterns and literal match rules.
- Updated ANSI rendering to show the menu bar, active menu row, optional line numbers, visible whitespace markers, and syntax styling while preserving selection/search priority.

## Known limitations

- Nano importer Tier 1 is intentionally narrow: no multiline `start=`/`end=`, header matching, comment metadata, or compatibility quirks.
- Syntax rules use simple byte-oriented literal matching rather than a full regex engine.
- File browser is a tested model; MVP-003 does not add destructive file actions, recent files, rename, copy, or delete.
- Whitespace display uses simple ASCII markers and never mutates document bytes.
- Cross-platform local verification on this machine proves macOS. Linux and Windows parity is maintained in code paths and CMake configuration, with full platform evidence expected from the CI matrix.
