# MVP-002 — Undo, Search, and Selection

## Objective

Make editing safe and useful beyond trivial changes.

## Scope

- undo/redo stack
- grouped text insertion undo
- preserve read/write mode rules for all editing actions
- find prompt
- find next/previous
- visible match highlight
- replace current
- replace all with confirmation
- selection model
- Shift+arrow selection
- cut/copy/paste internal clipboard
- cut line

## Acceptance criteria

- text edits can be undone/redone
- replace all is one undo group
- selection survives viewport movement
- deleting selected text is tested
- search highlights visible matches
- all MVP-002 behavior configures, builds, and tests on macOS, Linux, and Windows

## Implementation closeout notes

MVP-002 is implemented in core state with:

- undo/redo for recorded document replacements, including grouped printable insertion
- literal case-sensitive find with next/previous wrapping
- find prompt through `Ctrl+F`
- replace current through `Ctrl+R`
- replace all through `Ctrl+A` plus `y` confirmation
- replace-all recorded as one undo group
- selection stored in document coordinates
- Shift+arrow selection extension
- internal clipboard cut/copy/paste
- cut line through `Ctrl+K`
- visible search and selection highlighting in the ANSI renderer
- read-mode blocking for mutating MVP-002 actions

Known limitations:

- search is literal and case-sensitive
- regex, whole-word, and case-insensitive toggles remain later work
- clipboard is internal only; OS clipboard integration remains later work
- POSIX Shift+arrow support depends on terminal escape sequences and currently covers common `ESC [ 1 ; 2 A/B/C/D` forms
