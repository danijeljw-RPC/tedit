# MVP-001 — Single-File Raw Terminal Editor

## Objective

Build the first usable editor loop.

The editor should open a file, render it, allow cursor movement, edit text, and save.

## Scope

- POSIX raw terminal mode
- Windows console mode for accepted MVP behavior
- ANSI rendering
- command-line file open
- document line model
- read mode for existing files
- write mode entered with `w` / `W`
- cursor movement
- vertical scrolling
- per-line horizontal scrolling for active line
- UTF-8 byte-preserving text handling where practical
- insertion/editing only in write mode
- backspace
- delete
- newline insertion
- missing path creates a new unsaved document in write mode
- atomic save
- LF/CRLF line-ending preservation
- quit
- status bar
- basic tests for document, mode, save, and viewport behaviour
- CMake configure/build/test on macOS, Linux, and Windows

## Out of scope

- syntax highlighting
- selection
- search
- replace
- tabs
- menus
- spelling
- OS clipboard
- full Unicode grapheme/display-width perfection
- recovery files
- external modification detection

## Acceptance criteria

- `cmake -S . -B build` succeeds on macOS, Linux, and Windows
- `cmake --build build` succeeds on macOS, Linux, and Windows
- `ctest --test-dir build` succeeds on macOS, Linux, and Windows
- `./build/tedit file.txt` opens a file
- existing files open in READ mode
- pressing `w` or `W` enters WRITE mode
- editing is blocked in READ mode
- missing paths open as new WRITE-mode documents and save to the requested path
- arrow keys move cursor
- long active line scrolls independently
- inactive lines remain anchored at column 0
- active-line horizontal scroll resets when moving away from a long line
- typed characters modify the document
- LF and CRLF files preserve line endings after save
- Ctrl+S saves
- Ctrl+Q exits immediately and restores terminal
- macOS Option+Q exits immediately where the terminal sends a distinguishable input sequence

## Known starter gaps

The starter skeleton intentionally leaves some implementation incomplete:

- newline insertion is a TODO
- Windows terminal implementation must be brought to MVP parity before MVP-001 closes
- direct save must be replaced with atomic save before MVP-001 closes
- renderer is immediate ANSI output, not dirty-cell diffing yet
