# MVP-001 Closeout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close MVP-001 by replacing Windows placeholder terminal behavior, updating stale MVP documentation, and running final local verification plus an interactive smoke test.

**Architecture:** Keep the platform abstraction stable. POSIX continues using `termios` and escape sequences; Windows uses Console APIs to configure raw-ish input/output, read key events, query terminal size, and emit ANSI output through virtual terminal processing.

**Tech Stack:** C17, CMake 3.20+, POSIX `termios`, Windows Console API, existing in-repo `assert` tests.

## Global Constraints

- Existing files open in read mode.
- Missing command-line paths open as new unsaved documents in write mode.
- Editing is blocked in read mode.
- `Ctrl+S` saves.
- `Ctrl+Q` exits immediately.
- macOS `Option+Q` exits immediately where the terminal sends a distinguishable input sequence.
- Windows support for accepted MVP behavior must not remain placeholder-level.
- macOS, Linux, and Windows CMake configure/build/test paths must remain viable.
- No ncurses/PDCurses, GUI, plugins, Tree-sitter, LSP, or network functionality.

**Files:**

- Modify: `src/platform/terminal_win32.c`

**Interfaces:**
- Consumes: existing `Platform` API.
- Produces: Windows raw console mode setup and restoration.
- Produces: Windows key mapping for arrows, Home/End, Page Up/Page Down, Ctrl+Home/Ctrl+End, Delete, Backspace, Enter, Ctrl+S, and Ctrl+Q.
- Produces: Windows terminal size from `GetConsoleScreenBufferInfo`.

- [ ] **Step 1: Add Windows state fields**

Update `Platform` in `src/platform/platform.h` with Windows-only fields:

```c
#ifdef TEDIT_PLATFORM_WINDOWS
    void *input_handle;
    void *output_handle;
    unsigned long original_input_mode;
    unsigned long original_output_mode;
#endif
```

Update `platform_init` in `src/platform/platform.c` to initialize these fields to zero under `TEDIT_PLATFORM_WINDOWS`.

- [ ] **Step 2: Implement Windows Console API behavior**

Replace the placeholder `src/platform/terminal_win32.c` implementation with:

- `GetStdHandle` handles for input/output.
- `GetConsoleMode` preservation for both handles.
- input mode with line input and echo disabled.
- output mode with virtual terminal processing enabled when available.
- `ReadConsoleInputA` loop that ignores non-key events and key-up events.
- `KEY_EVENT_RECORD` mapping for accepted MVP keys.
- `WriteFile` for output.
- `GetConsoleScreenBufferInfo` for size.
- `SetConsoleMode` restoration on leave raw mode.

- [ ] **Step 3: Run local verification**

Run: `cmake -S . -B build && cmake --build build && ctest --test-dir build --output-on-failure`

Expected: all local macOS tests pass. Windows compile is covered by CI, not locally executable here.
